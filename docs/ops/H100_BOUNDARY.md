# H100 Boundary

## Purpose

H100 server verification is allowed only when it directly supports KORA AI Champion evidence closure.

## Allowed H100 Work

1. Verify whether expected evidence artifacts exist.
2. Inspect existing evidence outputs or summary logs.
3. Run bounded validation commands only if already documented or clearly safe.
4. Record only public-safe summary results in repo docs.

## Forbidden H100 Work

1. New broad GPU benchmarks.
2. Long-running experiments.
3. Server environment restructuring.
4. Dependency installation unless explicitly required and approved.
5. Production/customer/10x savings claims not backed by approved evidence.
6. Committing private server details.
7. Copying raw private logs into public docs.
8. Exposing hostnames, usernames, IP addresses, credentials, or raw private paths.

## Rule

If H100 verification is needed, summarize only the evidence outcome, not private access details.
